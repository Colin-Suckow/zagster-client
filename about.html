About

Zagmap is a mapping application using mapbox, node and express to display data from a bike sharing service

Client

Zagmap's client is a basic javascript page that uses http requests to retrieve data from the server. Mapbox's mapbox-gl-js library was used to display the map and the route lines. Routes are determined by using the start and end location of each bike ride, and feeding them into mapbox's bike directions api. The results from this call are then displayed on the map using mapbox-gl-js's layer system.

Server

Zagmap's server provides an http interface to a postgres database holding the bike data. The server is runs on top of node.js and uses express to handle requests. Both the server and database are hosted on heroku. Once a date is selected, zagmap client requests two data objects, the ride cordinates for a specific day, and the number of rides on those days. The server application interprets the query from the http request, and uses a standard sql query to retrieve and return the requested data. The server application is forked from Yong Bakos's implementation.
